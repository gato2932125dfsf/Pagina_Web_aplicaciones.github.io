¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite registrar los cambios en los archivos de un proyecto a lo largo del tiempo. Facilita la colaboración entre desarrolladores, mantiene un historial completo y permite revertir cambios si es necesario.

¿Por qué es necesario?

Git es fundamental para cualquier desarrollador, ya que permite controlar las versiones de un proyecto de manera eficiente. Evita la pérdida de trabajo, facilita la colaboración en equipo y permite revertir versiones anteriores si se comete un error.

Funcionalidades Principales

Ramificación (Branching)

Permite crear ramas de trabajo aisladas para experimentar con nuevas características sin afectar el código principal.

Historial de Versiones

Cada cambio se registra con un "commit", creando un historial completo y navegable del proyecto.

Colaboración Sencilla

Facilita la fusión de los cambios de diferentes desarrolladores de manera controlada y sin conflictos.

Reversión de Cambios

Puedes deshacer cambios o volver a una versión anterior del proyecto en cualquier momento.

Funcionamiento de Git

  1. Paso 1: Clonar el repositorio
    git clone https://github.com/tu-usuario/nombre-del-repo.git
  2. Paso 2: Navegar al directorio
    cd nombre-del-repo
  3. Paso 3: Agregar cambios
    git add .
  4. Paso 4: Guardar cambios (Commit)
    git commit -m "feat: Agrega un nuevo archivo"
  5. Paso 5: Enviar cambios (Push)
    git push origin main
